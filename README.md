# DSRPracticeChat

Интернет-чат, реализованный на языке C. Приложение представляет собой консольный клиент-серверный чат с поддержкой групп, модерации сообщений и мониторинга активных сессий.

## Возможности

**Пользовательские возможности:**
- Подключение/отключение общего чата;
- Создание новых групп;
- Передача файлов группе/пользователю;
- Постоянная запись чата в файл;
- Изменение своих сообщений.

**Возможности сервера:**
- Перезапуск сервера с сохранением всех настроек;
- Постоянное логирование всех сигналов и сообщений в системе;
- Индикация всего списка сессий и чатов.

## 1. Сборка и запуск системы

### Требования

- Компилятор C (`gcc`);
- Утилита `make`;
- Библиотека **libwebsockets** — обязательна для сборки и работы сервера и клиента.

### Установка зависимостей

Установите `libwebsockets` через пакетный менеджер вашей системы.

Debian / Ubuntu:

```bash
sudo apt update
sudo apt install libwebsockets-dev
```

Fedora:

```bash
sudo dnf install libwebsockets-devel
```

Arch Linux:

```bash
sudo pacman -S libwebsockets
```

macOS (Homebrew):

```bash
brew install libwebsockets
```

Затем склонируйте репозиторий:

```bash
git clone https://github.com/kotovro/DSRPracticeChat.git
cd DSRPracticeChat
```

### Сборка

Сборка выполняется из корня проекта через `make`. Доступны следующие цели:

```bash
make          # то же, что make all — собирает клиент и сервер
make all      # собирает и клиент, и сервер
make client   # собирает только клиент
make server   # собирает только сервер
make clean    # удаляет артефакты сборки клиента и сервера
```

Makefile в корне делегирует сборку вложенным Makefile'ам в каталогах `client/` и `server/` (`$(MAKE) -C client`, `$(MAKE) -C server`).

### Запуск сервера

```bash
./server [-p server_port] [-d logs_directory]
```

| Флаг | Аргумент | Назначение |
|------|----------|-----------|
| `-p` | `server_port` | Порт, на котором запускается сервер. Допустимые значения: от 1 до 65535. При некорректном значении сервер завершает работу с ошибкой. |
| `-d` | `logs_directory` | Директория для записи логов. Если указанной директории не существует, логи пишутся в директорию по умолчанию. |
| `-h` | — | Вывод справки по использованию. |

Пример:

```bash
./server -p 8080 -d ./logs
```

### Запуск клиента

```bash
./client [-s server_ip] [-p server_port] [-d logs_directory]
```

| Флаг | Аргумент | Назначение |
|------|----------|-----------|
| `-s` | `server_ip` | Адрес сервера, к которому подключается клиент. |
| `-p` | `server_port` | Порт сервера. Допустимые значения: от 1 до 65535. При некорректном значении клиент завершает работу с ошибкой. |
| `-d` | `logs_directory` | Директория для записи логов. Если указанной директории не существует, логи пишутся в директорию по умолчанию. |

Пример:

```bash
./client -s 127.0.0.1 -p 8080 -d ./logs
```

## 2. Инструкция пользователя чата

Приложение является **консольным**: все действия выполняются вводом текста и команд в терминале. Отправка сообщения и выполнение команды происходят по нажатию **Enter**.

### Регистрация

Чтобы создавать группы и редактировать сообщения, необходимо зарегистрироваться:

```
/login <имя клиента>
```

Требования к имени клиента:
- не должно совпадать с именем другого уже залогиненного клиента;
- не может быть `Общий чат` или `Сервер` (эти имена зарезервированы во избежание недопонимания);
- не может совпадать с названием уже существующей группы.

### Отправка сообщений в общий чат

Чтобы написать в общий чат, просто наберите текст сообщения и нажмите **Enter**.

### Создание группы

```
/add_group <название группы>
```

Требования к названию группы аналогичны требованиям к имени клиента: название не может быть `Общий чат` или `Сервер`, не должно совпадать с именем залогиненного клиента и с названием уже существующей группы.

### Редактирование своего сообщения

```
/edit <guid сообщения> <новый текст сообщения>
```

Аргументы:
- `<guid сообщения>` — идентификатор сообщения, которое нужно изменить;
- `<новый текст сообщения>` — новый текст, который заменит исходный.

### Прочие возможности

- Передача файлов группе или отдельному пользователю;
- Отключение и повторное подключение к общему чату;
- Постоянное сохранение переписки в файл;
- Сжатие траффика.
